# ShopERP — Documentation

This directory contains user-facing and developer-facing documentation for the ShopERP system.

## Structure

```
docs/
├── README.md              ← you are here
├── api/                   ← generated API documentation (after modules are built)
└── modules/               ← per-module documentation (after modules are built)
```

## Documentation Lifecycle

Documentation is generated by the Documentation Agent after each module is completed and verified. It is based on the actual implementation, not the knowledge-base specifications.

The Documentation Agent audits all documentation for accuracy at every phase boundary.

## Source of Truth Hierarchy

1. **Source code** — always the ground truth for how the system behaves
2. **knowledge-base/** — the intended specification (may drift from code; Documentation Agent detects drift)
3. **docs/** — user-facing documentation derived from source code
4. **.ai/work-trail/** — historical record of what was built and why

If any of these conflict, the source code is correct and the others must be updated.
